# Screenshots guide — BuzzOff

Google Play requires **at least 2 phone screenshots** (max 8). Use a phone or emulator; recommended size **1080×1920 px** (9:16) or follow [Play’s specs](https://support.google.com/googleplay/android-developer/answer/9866151).

## Suggested screens (in order)

1. **Country picker** — “Select your country” with the flag grid (Israel, Germany, France, UK, etc.). Shows the first step.
2. **Download progress** — “Downloading [Country]…” with the progress bar. Shows the one-time pack download.
3. **Map + alert** — Map with your position, camera markers, and the “Speed camera — XXX m” pill. Best with “Monitoring cameras” in the status bar. Shows the core value.
4. **Map idle** — “Waiting for driving…” so users see it runs in the background until they drive.
5. **Settings** — Settings screen with Alert distance, Alert type, Activate at speed, Sleep when stopped, Country. Shows customization.

## How to capture

- **Device:** Run the app on a real phone, go to each screen, take a screenshot (e.g. power + volume down).
- **Emulator:** Run the app in Android Emulator, open each screen, use the camera icon in the emulator toolbar or the device shortcut.
- **Optional:** Add a short caption or device frame in an image editor; keep text readable and avoid crowding.

## Tips

- Use a **clean state**: no dev menu or debug overlays.
- Prefer **day/light theme** for consistency (or match your store listing).
- If you add text overlays, keep them short (e.g. “Pick your country”, “Offline maps”, “Silent alerts”).
